# Page Family Checks

## LIST

- filters align with canonical list screen
- result actions are in the same position
- excel/create buttons follow sibling rules

## DETAIL

- lookup context strip exists if sibling detail screens use it
- read-only sections use the same spacing

## EDIT and CREATE

- field controls use shared inputs
- bottom primary button matches sibling forms

## APPROVE

- review action area matches sibling approval screens
- modal footer button widths are consistent

## AUTHORITY

- role selectors and permission toggles use shared controls
- bottom action area matches authority siblings
