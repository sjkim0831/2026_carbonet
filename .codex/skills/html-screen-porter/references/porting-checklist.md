# Porting Checklist

Before finishing an HTML-to-React port, verify:

- page type is explicitly chosen
- screen family is explicitly chosen
- route definition exists
- page manifest exists
- page registry exists
- menu context rule is clear
- hidden screens have active menu mapping
- no raw `input`, `select`, or `button` remain without justification
- status messages use the shared pattern
- bottom action buttons match sibling screens
- input heights match sibling screens
- assumptions from the HTML source are documented
