# Module Selection Checklist

Generated on 2026-03-21 for Resonance screen-generation module attach safety.

## Goal

Provide the repeatable checklist used before moving from module selection to scaffold/build.

## Checklist

Before continuing, confirm:

1. project and scenario are selected
2. required modules are selected
3. high-impact modules were reviewed through install popup
4. dependency resolution is green
5. style coverage is green
6. style dedupe is green
7. backend and DB impact review is green
8. rollback readiness is green
9. runtime package preview reflects the selected modules
10. selected modules do not violate common/project ownership boundaries
