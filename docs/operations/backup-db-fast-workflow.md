# Backup DB Fast Workflow

이 문서는 백업 DB 서버에 붙어 작업할 때 느린 GUI 경로 대신 빠른 작업 경로를 기본 원칙으로 명시한다.

## 원칙
- 백업 DB 서버에서 단순 조회/수정/점검은 `mosh + tmux + dbsql` 경로를 기본으로 사용한다.
- DBeaver는 확인용으로만 제한적으로 사용한다.
- 새 SSH 세션을 매번 열어 한 줄 명령만 치는 방식보다, 지속 세션을 유지하는 방식을 우선한다.

## 대상 서버
- Host: `34.82.141.193`
- DB Port: `33000`
- DB Name: `carbonet`

## 권장 접속 순서
```bash
mosh sjkim08311@34.82.141.193
tma
dbsql
```

짧은 확인:
```bash
dbsql -c "select count(*) from dba.comtnmenuinfo"
```

## 이유
- `mosh`: 네트워크 흔들림과 지연에 강함
- `tmux`: 세션 유지, 긴 작업 보존
- `dbsql`: 로컬 CUBRID 경로로 직접 실행, GUI 메타데이터 왕복 제거

## 금지하지는 않지만 비권장
- DBeaver로 처음 테이블을 여는 방식
- 새 SSH 연결을 매번 새로 열어 단발 조회하는 방식

## 현재 확인된 체감 기준
- 새 SSH 연결 포함 단순 조회: 약 `2.3초`
- 서버 내부 `dbsql` 단순 조회: 약 `86ms`
- 실제 `mosh -> tmux -> dbsql` 실행 시 DB 내부 쿼리 시간: 약 `0.0066초`

## 앱 작업 시 지침
- 앱에서 백업 DB를 붙여 확인/수정할 때는 위 빠른 경로를 우선 사용한다.
- 백업 DB는 운영 대체보다 복구/검증/긴급 확인용이다.
- 동시 작업자는 적게 유지한다. 1GB VM 기준으로 `1~2명` 안정, `3명` 이내 가벼운 작업 권장.
