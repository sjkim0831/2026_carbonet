Design Baseline

홈 화면과 관리자 화면이 같은 업무 메타데이터를 보도록 설계하는 기준 시안

이 시안은 단순 랜딩 페이지가 아니라, 홈 대시보드에서 보이는 카드와 관리자 화면에서 관리하는 업무 항목이 같은 구조를 바라보게 만드는 기준안입니다. 이후 Stitch, HTML 포팅, React 구현, 관리자 생성 화면 설계까지 같은 언어로 이어지게 하는 출발점입니다.

홈 메뉴와 관리자 메뉴의 대응 구조

실서비스에서는 홈에서 보이는 모든 업무 항목이 관리자에서 생성, 수정, 통제, 감사 가능해야 합니다. 홈 카드와 관리자 메뉴를 같은 메타 모델로 묶는 방식이 기준입니다.

대표 메뉴 매핑표

홈/업무 화면 사용자 액션 대응 관리자 메뉴 관리 범위
배출지 홈 대시보드 핵심 배출지 보기, 우선업무 진입, 신규 등록 배출지 마스터 관리 배출지 기본정보, 책임자, 상태, 노출 우선순위, 핀 고정 여부
데이터 입력 화면 배출량 입력, 월별 수정, 산정 실행 월별 데이터 통제 관리 입력 마감, 필수 항목, 검증 규칙, 재오픈 정책, 이력 확인
증빙 업로드 화면 증빙 파일 등록, 교체, 제출 증빙 요청 관리 필수 파일셋, 버전, 반려 사유, 만료 정책, 다운로드 권한
검증 현황 화면 검증 단계 확인, 피드백 대응, 결과서 확인 검증 진행 관리 검증 단계, 외부기관 상태, 승인/반려, 결과서 이력, 담당자 배정
분석 리포트 화면 이상징후 확인, 목표 이탈 원인 확인 임계치·알림 정책 관리 알림 기준값, 예외 규칙, 대상자, 노출 위치, 후속 업무 자동 생성

실서비스 누락 방지용 운영 구조

화면 시안만으로 끝나지 않도록, 업무 생성부터 이력·배포·운영까지 빠지지 않게 설계해야 하는 필수 흐름입니다.

01 업무 정의 메뉴, 배우(actor), 상태, KPI, 필수 증빙, 담당 화면 정의
02 화면군 설계 홈, 목록, 상세, 입력, 검증, 승인, 관리자 관리 화면 동시 설계
03 계약 고정 페이지 ID, API, DTO, 상태코드, 권한, 알림 규칙, 도움말 anchor 고정
04 구현 및 추적 React 페이지, 관리자 API, 감사로그, trace, manifest, registry 반영
05 운영 검증 누락 메뉴, 예외 상태, 권한 차단, 반려 재처리, 배포 후 운영 흐름 확인

이 기준안에서 파생될 화면 패밀리

Dashboard 홈 대시보드, 핵심 요약, 우선업무 큐, 운영 가이드 레일
List 배출지 목록, 업무 큐 목록, 증빙 요청 목록, 검증 진행 목록
Detail / Edit 배출지 상세, 월별 데이터 편집, 증빙 상세, 검증 결과 상세
Approve / Authority 승인 처리, 상태 변경, 권한/노출 정책, 알림 규칙 관리

시안 사용 원칙

원칙 1 홈과 관리자가 서로 다른 데이터를 따로 가지지 않음
원칙 2 버튼 하나마다 후속 화면, 상태 변화, 관리자 제어점이 있어야 함
원칙 3 설계 단계에서 페이지 ID, 업무코드, 상태코드, 관리메뉴 코드를 먼저 확정
원칙 4 도움말, 감사이력, 추적 메타데이터를 화면 이후가 아니라 설계 시점에 포함
